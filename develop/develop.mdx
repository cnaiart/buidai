# 本地部署与开发

<div id="overview">
  ## 概述
</div>

本地部署项目以启动开发模式服务器

<div id="local-deployment">
  ## 本地部署
</div>

上一篇《[前置准备](/ai-tools/Preposition)》我们已经完成了环境的准备，接下来我们来本地部署和启动必定AI。

<div id="clone-repository">
  ### 拉取代码
</div>

首先从代码仓库拉取最新代码：

```bash theme={null}
# Github
git clone https://github.com/BidingCC/BuildingAI.git
# or Gitee
git clone https://gitee.com/BidingCC/BuildingAI.git
```

<div id="pull-docker-images">
  ### 拉取 Docker 镜像
</div>

确保已安装 Docker，若未安装请先安装 Docker。

```bash theme={null}
cd BuildingAI
docker compose up -d
```

等待几分钟拉取镜像（取决于网络速度）。

<Info>
  **提示**：镜像拉取过程可能需要几分钟时间，具体取决于您的网络速度。
</Info>

<div id="check-deployment-progress">
  ### 查看部署进度
</div>

拉取完成之后，可以通过以下命令查看部署进度:

```bash theme={null}
# 单次查看
docker logs buildingai-nodejs
# 实时查看
docker logs -f buildingai-nodejs
# 查看最近100行
docker logs --tail 100 buildingai-nodejs
# 查看最近100行并实时跟踪
docker logs --tail 100 -f buildingai-nodejs
```

直到终端打印出如下内容则说明部署成功：

```
[SUCCESS] BuildingAI has been successfully deployed!
[INFO] Server is running on http://localhost:4090
```

此时可以访问 http://localhost:4090/install 进行初始化安装

<Card title="初始化安装" icon="settings" href="http://localhost:4090/install">
  访问初始化安装界面
</Card>

<div id="stop-docker-container">
  ### 停止 Docker 容器
</div>

初始化安装完成之后，为了便于开发，可以停止或者删除 Docker 中 buildingai 容器下的 buildingai-nodejs 容器，使用我们本地安装的Node.js来运行项目。

```bash theme={null}
# 停止容器
docker stop buildingai-nodejs
```

<div id="remove-docker-container">
  ### 删除 Docker 容器
</div>

删除需要在容器停止之后才能删除：

```bash theme={null}
# 删除容器
docker rm buildingai-nodejs
```

<div id="local-development">
  ## 本地开发
</div>

接下来我们可以使用本地的 Node.js 环境来运行项目进行开发。

```bash theme={null}
# 进入项目目录
cd BuildingAI
# 安装依赖
pnpm install
# 启动后端服务
cd packages/api && pnpm dev
# 启动前端服务（另起一个终端）
cd packages/web && pnpm dev
```

<Card title="访问前端界面" icon="browser" href="http://localhost:4091">
  通过本地地址 http://localhost:4091 访问前端界面
</Card>

<Info>
  **开发提示**：前端和后端服务需要在不同的终端中运行，确保两个服务都已启动才能正常使用应用。
</Info>
